# Решение по корневому домену `ingwaz.space`

**Проект:** Ingwaz (публичное имя витрины: **«Сигнал аудитории»** — см. [NAMING-AND-POSITIONING.md](./NAMING-AND-POSITIONING.md)).  
**Статус:** зафиксировано по итогам разборов (Cursor + альтернативная позиция ChatGPT, май 2026).  
**Пересмотр:** через **30** и **90** дней по метрикам в §6 или при смене primary-продукта.

## 1. Контекст

- Зона **`ingwaz.space`**: продукты на **поддоменах** (отдельный деплой, канон, Вебмастер, sitemap).
- **Корень** `https://ingwaz.space` **не задействован** под продукт (код — репозиторий **Ingwaz**).
- Фаза портфеля: **прототипы в проде** → индексация, клики, активации, первые оплаты (не полноценный PMF).
- Ограничения: **solo-founder**, ~**2–4** качественных SEO-материала в месяц максимум, общий стек Laravel + React.

**Диагноз:** на каждом поддомене мало URL (4–6 индексируемых); SEO растёт медленно. Идея «корень потянет весь SEO на subs» **не работает** без большого объёма контента и времени; subdomain ≠ subdirectory, переносимого authority почти нет.

---

## 2. Решение (что делаем)

### 2.1. Роль `ingwaz.space` (корень)

**Минимальный thesis-сайт**, не медиа-портал и не мультирубрика.

| Делаем | Не делаем |
|--------|-----------|
| 1 страница **thesis** + карта инструментов (ссылки на поддомены) | Рубрики `/seo/`, `/marketing/`, `/creators/` «на будущее» с 2 статьями |
| Опционально **0–3** коротких материала под thesis (не отдельные «журналы») | 50+ статей «для индекса» |
| Юридические/технические страницы при необходимости | Дубли коммерческих лендингов subs |
| UTM / Метрика: клики **с корня на subs** | Ожидание, что корень даст основной organic-трафик |

**Thesis (рабочая формулировка):**

> Инструменты **audience intelligence** для авторов экспертного контента — от feedback аудитории к решениям по контенту.  
> (Якорь: **CommentSignal**; остальные продукты — смежные утилиты, не «всё для всех».)

**Публичное название:** **Сигнал аудитории** (смысл на русском в title/meta, не голый `ingwaz`).

### 2.2. Где основной SEO

**Product-led SEO на поддоменах** — приоритет **1–2** продукта:

| Роль | Продукт | Поддомен (канон) |
|------|---------|------------------|
| **Primary** | CommentSignal | `commentsignal.ingwaz.space` |
| **Secondary** (опционально, если есть время) | SEO Brief *или* Competitor Snapshot | `seobrief.*` / `compsnap.*` |
| **Maintenance** | Landing Critic, Invoice, Quiz | свои subs; без новых фич, без контента на корне под них |

Типы страниц на primary/secondary (не «блог»):

- use-case / JTBD («анализ комментариев YouTube для …»)
- **examples** с артефактами (обезличенный output, скрин)
- comparison / «vs ручной процесс»
- узкие intent под beachhead (expert + long-form YouTube)

**Quality bar:** страница без уникального артефакта или конкретного сценария — **не публикуем** (риск thin content).

### 2.3. Creator-линия и сегменты блогеров

См. [BLOGGER-SEGMENTS.md](./BLOGGER-SEGMENTS.md). Кратко: **не** все 9 типов на корне; фокус **YouTube long-form + экспертные**; Telegram — позже.

### 2.4. Миграция subdomain → subdirectory

**Не делаем** на текущей стадии: нет authority, есть риск инженерного отвлечения, PMF не доказан.

## 3. Отвергнутые варианты (кратко)

| Вариант | Почему не выбран сейчас |
|---------|-------------------------|
| **A) B2B hub** на корне (маркетинг, конкуренты, SEO) | Нет единого ICP; generic SaaS-blog; B2B-контент остаётся на **subs** |
| **B) Creator-only hub** (медиа только для блогеров) | Сильный narrative, но без объёма = слабый SEO; допустим **узкий thesis**, не журнал |
| **C) Мультирубрика** (SEO / маркетологи / блогеры / launch) | При 2–4 статьях/мес — dilution; «кладбище рубрик» для solo |
| **Медиа на корне как главный канал** | Увеличивает latency обратной связи по PMF; риск productive procrastination |

**Отложено (пересмотр после сигнала с product-pages):**

- Одна рубрика на корне **только** под thesis «audience intelligence», 10+ материалов в silo
- Полноценный **E) thesis-hub**, отсекающий часть B2B из narrative (мысленный эксперимент, не ребрендинг всего портфеля)

---

## 4. Anti-goals (явные запреты)

1. **Не** открывать 3–4 рубрики с пустыми разделами «для красоты».
2. **Не** публиковать статьи на корне, которые дублируют intent product-лендингов subs (каннибализация).
3. **Не** ставить на корне 6 равноправных CTA «попробуйте всё».
4. **Не** гнать programmatic-страницы без уникального output (шаблонные AI-страницы).
5. **Не** расширять портфель новыми деплоями, пока primary не даёт **повторяемый** сигнал (activation / оплата / weekly reuse).
6. **Не** тратить >**8 ч** на одну «SEO-статью», если нет гипотезы измеримого перехода на tool.
7. **Не** позиционировать `ingwaz` как «коллекцию несвязанных SaaS» в H1 корня — только thesis + инструменты.

## 5. План 90 дней (ориентир)

### Месяц 1

- **Корень (репо Ingwaz):** thesis-страница + ссылки на продукты; sitemap 3–5 URL; редиректы http/www при необходимости.
- **CommentSignal:** **15–20** intent/examples-страниц (не лонгриды «как вести канал»).
- **Secondary sub** (если выбран): до **10–15** страниц — только при сохранении темпа на primary.

### Месяц 2

- Доработка страниц с лучшим CTR; 5–10 новых intent-URL на primary.
- Distribution вне SEO: нишевые комьюнити (Telegram, Reddit где уместно, creator-чаты) — по продукту, не «блог ingwaz».

### Месяц 3

- Решение go/no-go по §6.
- Если organic → signups есть: рассмотреть **одну** узкую контент-ветку на корне (3–5 материалов max).
- Если нет: **не** открывать медиа на корне; удвоить examples на `commentsignal.*`.

## 6. Метрики и пороги

### 6.1. Через 30 дней (пересмотр тактики)

**CommentSignal (primary):**

| Метрика | Сигнал «продолжаем» | Сигнал «меняем гипотезу» |
|---------|---------------------|---------------------------|
| Новые intent-URL в индексе | ≥ **10** полезных страниц | < 5 |
| Organic impressions (сумма новых URL) | ≥ **500** / мес (ориентир) | почти 0 |
| Переходы на signup/import с organic | ≥ **10** | 0 |
| Activation (import + insights) | ≥ **3** пользователя | 0 |

**Корень:**

| Метрика | Сигнал «продолжаем thesis-сайт» | Сигнал «корень не трогаем» |
|---------|-----------------------------------|----------------------------|
| Клики на subs с корня | ≥ **20** / мес | < 5 |
| Время на поддержку корня | < **4 ч** / мес | ощущение «второй продукт» |

*Абсолютные пороги гибкие; важнее тренд и signups, не vanity impressions.*

### 6.2. Через 90 дней (go/no-go медиа на корне)

**Продолжаем product-led SEO на subs; корень остаётся минимальным**, если:

- CommentSignal: есть **повторяемые** signups или оплаты с organic/direct intent-страниц **или**
- Чёткий канал distribution (не только «ждём SEO»)

**Открываем одну контент-ветку на корне** (не мультирубрику), только если **дополнительно**:

| Метрика | Go (stretch) | No-go |
|---------|--------------|-------|
| Indexed useful pages (subs + корень) | ≥ **30** | < 15 |
| Organic clicks to tools (сумма) | ≥ **100** / мес | < 30 |
| Signups attributed to SEO pages | ≥ **15** / мес | < 5 |
| Signup conversion (SEO → регистрация) | ≥ **2%** | < 1% |
| Среднее время на SEO-страницу | ≤ **8 ч** | > 15 ч |

**Жёсткий критерий ценности:** материал / страница без signups, returning users или qualified traffic — **luxury**; не масштабируем.

### 6.3. Что заставит пересмотреть всё решение

- Другой продукт даёт **10×** лучшую конверсию с organic при тех же усилиях → смена primary.
- Пользователи **сами** называют продукты «экосистемой» и просят единый вход (интервью, не гипотеза).
- Одна экспериментальная рубрика на корне: **>3k impressions**, **>100 clicks**, **>5 signups** за 60 дней **без** ручного продвижения каждой статьи.
