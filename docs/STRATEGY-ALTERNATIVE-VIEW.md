# Альтернативная позиция (внешний разбор, май 2026)

Сжатое изложение ответа ChatGPT на промт «оспорить стратегию корня + subs». Не заменяет [ROOT-DOMAIN-DECISION.md](./ROOT-DOMAIN-DECISION.md); фиксирует аргументы «против» для пересмотра.

## Executive summary (чужая позиция)

- Дефицит не domain authority, а **концентрация сигнала**; 6+ направлений при solo — распыление.
- SEO на корне **не заменяет PMF**; риск productive procrastination (статьи вместо signups).
- **Мультирубрика (C)** — слабый выбор на старте; **product-led SEO на subs (D)** — default.
- Сильнейший wedge портфеля: **CommentSignal / audience intelligence**.
- Корень: либо **узкий thesis-hub**, либо **не трогать** (about + links).

## Что приняли в наше решение

- Минимальный корень, не медиа-портал.
- Primary SEO на `commentsignal.*` (intent + examples + артефакты).
- Anti-goals против мультирубрики и thin pages.
- Метрики 30/90 с фокусом на signups, не vanity impressions.

## Что смягчили

- Жёсткие KPI ChatGPT (10k impressions / 90 дней) — только как stretch.
- «Заморозить 4 продукта» → режим **maintenance**, не закрытие.
- Полный **thesis-hub (E)** с отсечением B2B — отложен.

## Варианты, которые мы не выбрали как старт

| Код | Суть |
|-----|------|
| A | B2B hub на корне |
| B | Creator-only медиа-hub |
| C | Мультирубрика по ролям |
| D (как единственный фокус) | Только subs, корень не трогать вообще |

## Пересмотр позиции

Через **30 дней** — см. [ROOT-DOMAIN-DECISION.md](./ROOT-DOMAIN-DECISION.md) §6.1.  
Если одна экспериментальная ветка на корне даст **>100 clicks и >5 signups** за 60 дней без ручного пуша каждой статьи — расширять контент на корне.
